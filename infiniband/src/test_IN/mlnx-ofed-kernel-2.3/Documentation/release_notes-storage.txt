3. Supported Platforms and Operating Systems
===============================================================================
Supported platform is x86_64.
Requires MLNX-OFED-2.3-1.0.0

Full Cycle:
    RHEL7.0: iSER RoCE, iSER IB, SRP
    RHEL7.0 INBOX*: iSER RoCE, iSER IB, SRP
    RHEL6.5: iSER IB, iSER RoCE, SRP
    RHEL6.4: iSER IB, iSER RoCE, SRP
    Sles11SP3: iSER IB, iSER RoCE, SRP

Partial Cycle:
    RHEL6.3: iSER IB, iSER RoCE, SRP
    OEL6.4 (UEK kernel): iSER IB, SRP
    OEL6.5 (UEK kernel): iSER IB, iSER RoCE, SRP
    Sles12: iSER IB, iSER RoCE, SRP
    Sles11SP2: iSER IB, iSER RoCE, SRP
    Sles11SP1: iSER IB, iSER RoCE, SRP
    Sles10SP3**: iSER IB (Limited support)
    Ubuntu12.04.4 (kernel 3.11): iSER IB, iSER RoCE, SRP
    Ubuntu14.04: iSER IB, iSER RoCE, SRP
    FC19: iSER IB, iSER RoCE, SRP
    FC20: iSER IB, iSER RoCE, SRP


* INBOX means the distro RDMA stack, without the need to install MLNX_OFED.
** Same iSER as in 1.2 (no code change).

5. Resolved Issues
===============================================================================
iSER:
 - Kernel crash in debug kernel in disconnect flow due to memory access after
   free.

 - In out-of-memory condition iSER might leak the receive-CQ memory
   allocations.

 - Connect-IB: Connection could fail allocating resources during stress of
   logins and logouts.

 - IO to a DIF enabled LUN may generate a transient error completion (will
   recover with connection re-establishment).

 - DIF type 2,3 and DIF types 1,2,3 with IP_CSUM generate PI errors on
   unwritten blocks.
   workaround:
   Upgrade FW to 10.10.4000.

6. Known Issues
===============================================================================
SRP/iSER:
 - RHEL7.0: rescan-scsi-bus.sh doesn't rediscover provisioned LUNs
   workaround:
   - Use older version of the script from RHEL6.4
   - iSER: Use "iscsiadm -m session --rescan"


SRP:
 -  The driver is tested with Storage target vendors recommendations for
    multipath.conf extensions (ZFS, DDN, TMS, Nimbus, NetApp).

 - MLNX_OFED SRP installation breaks ibmvstgt and ibmvscsi symbol resolution
   in RHEL7.0

 - When deleting non default pkey during targets discovery SRP may generate a
   call trace and can be ignored.

 - In high stress IO with unstable links SRP Initiator may generate a Call
   trace that can be safely ignored.

 - Ubuntu: Setting multipath failback value to 'followover' in multipath.conf
   may cause mutipathd to not restore paths properly and IO may break.

 -  SRP interop known issues:

    * DDN Storage Fusion 10000 target
      - DDN does not accept non-default P_Key connection establishment.

    * Oracle Sun ZFS storage 7420
      - ZFS does not accept non-default P_Key connection establishment.
      - Occasionally the first command to a LUN may not be serviced,
        aborted, and cause a successful re-connection to the target.

    * Ungraceful power cycle of an initiator connected with Targets DDN, Nimbus, NetApp
      may result in temporary "stale connection" messages when initiator reconnects.


iSER:
 -  On SLES, the ib_iser module does not get loaded on boot.
    workaround:
    Add a dummy interface using iscsiadm:
    - iscsiadm -m iface -I ib_iser -o new
    - iscsiadm -m iface -I ib_iser -o update -n iface.transport_name -v ib_iser

 - Ubuntu12 requires updating the user space open-iscsi to v2.0.873

 - Initiator does not respect interface parameter while logging in
   workaround:
   config each interface on different subnet.

 - ZFS appliance: connection establishment occurs twice which may cause iSER
   to log a stack trace.

 - iscsid may hang if target crashes during logout sequence.
   reproducible with TCP.

 - iscsid v2.0.873 can enter an endless loop on bind error.
   fix pending:
   https://groups.google.com/forum/#!searchin/open-iscsi/Fix$20infinite$20loop$20when$20conn$20/open-iscsi/zgLXgf28LVE/e3qkz8ZzwHMJ

 - SM LID reassign during traffic on OEL6.4 uek kernel with virtual function
   generates soft lockup trace.

 - DIF: When running IO over FS over DM during unstable ports, block layer
   BIOs merges may cause false DIF error.
   workaround:
   disable merges: echo 2 > /sys/block/$DEV/queue/nomerges
